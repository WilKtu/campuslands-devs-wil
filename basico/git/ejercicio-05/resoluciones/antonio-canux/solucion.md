# Push de Rama - Sistema de Taller de Motos

**Camper:** Antonio Canux

## Descripción

Este ejercicio consistió en crear una solución en una rama personal y publicarla en el repositorio remoto utilizando Git y GitHub.

## Objetivo

Subir correctamente una rama de trabajo al repositorio remoto, verificando que los cambios queden disponibles en GitHub.

## Procedimiento Realizado

### 1. Verificación de la rama actual

Se verificó que el trabajo se estuviera realizando sobre la rama personal mediante:

```bash
git branch
```

### 2. Verificación de cambios

Se comprobó el estado del repositorio:

```bash
git status
```

### 3. Preparación de archivos

Se agregaron los archivos modificados al área de preparación:

```bash
git add .
```

### 4. Creación del commit

Se registraron los cambios realizados mediante un commit:

```bash
git commit -m "feat: estructura inicial sistema de taller de motos"
```

### 5. Envío de la rama al repositorio remoto

Se publicó la rama en GitHub utilizando:

```bash
git push -u origin alumno/antonio-canux/ejercicio-05
```

### 6. Verificación en GitHub

Se verificó que la rama estuviera disponible en el repositorio remoto consultando la sección de ramas en GitHub.

## Evidencia
### URL del repositorio

```
https://github.com/canuxantonio502/campuslands-devs
```

## Resultado

La rama fue creada, los cambios fueron confirmados mediante un commit y posteriormente publicados exitosamente en GitHub, quedando disponibles para futuras integraciones mediante Pull Request.
